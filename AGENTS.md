## Learned Workspace Facts

- MediaBubble is an Nx monorepo: market apps `apps/web-eg` (Egypt) and `apps/web-ae` (UAE), plus `apps/brand` and internal ops app `apps/launcher` (port 3003, `pnpm run dev:launcher`); shared packages include `packages/shared` and `packages/design-system`. Planning docs live under `docs/` (`docs/CONTEXT.md` is the master AI handoff; launcher specs at `docs/launcher/`, not `docs/app/launcher/`).
- Shared CSP/security headers and middleware helpers live in `packages/shared/*.cjs` wired from market apps' and brand's `middleware.ts` and `next.config.js`; import CSP via `@mediabubble/shared/csp-middleware`. **MediaBubble Launcher** uses `proxy.ts` for JWT route gating, not `middleware.ts` (Next.js 16 builder). `export const config.matcher` must be an inlined literal array in each `middleware.ts` (constants imported from `.cjs` cannot be statically analyzed and break dev bundles). App `tsconfig.json` files that override `paths` must repeat shared subpath aliases from `tsconfig.base.json` or typecheck fails in middleware. Cross-Nx-project imports (e.g. Tailwind `mbPreset`) must use package subpaths like `@mediabubble/design-system/tailwind-preset`, not relative `packages/` paths—required for `enforce-module-boundaries` lint. `getCspNonce()` in `packages/shared/src/csp-nonce.ts` is async—root layouts in `web-eg`, `web-ae`, and `brand` must `await` it because Next `headers()` is Promise-based in current types/CI.
- Dark mode is class-based (`html.dark`) via shared `ThemeProvider` from `@mediabubble/shared/client` with storage key `mediabubble-theme`; inject `THEME_INIT_SCRIPT` in each app layout `<head>` to prevent flash. `web-eg`, `web-ae`, and `apps/brand` share this stack; `data-theme` is not used.
- Semantic `brand-*` tokens (`bg-brand-surface`, `text-brand-text-muted`, etc.) map to CSS variables in each app's `globals.css` and support `dark:` variants; fixed hex tokens (navy, yellow, blue, red) stay constant across themes.
- Root `package.json` exposes `pnpm run dev:brand` (port 3002), `dev:eg:clean`, `dev:ae:clean`, and `dev:launcher:clean` (port 3003) to reset dev (kill port, wipe `.next`/cache, stale PWA workers on market apps, restart); use after webpack `Cannot read properties of undefined (reading 'call')`, Playwright leaving a stale launcher server on :3003, or overlapping prod builds—avoid production builds while dev is running. In the browser: hard refresh, Incognito, or clear site data for `localhost` if the error persists after a clean restart. Local dev uses root `pnpm` only—do not run `npm install` inside `apps/*` (creates stray per-app `node_modules` and `package-lock.json`); pnpm expects root `pnpm-workspace.yaml` (`packages: apps/*`, `packages/*`)—root `package.json` `workspaces` remains for npm CI. CI installs with `npm ci` on Node 22; `nx.json` sets `cli.packageManager: "npm"` because committed `pnpm-lock.yaml` makes Nx auto-detect pnpm and fail without `node_modules/.modules.yaml`—CI also drops `pnpm-lock.yaml` in the runner as a guard. After adding dependencies to any workspace `package.json`, run `npm install` at the repo root and commit `package-lock.json` in the same PR—Vercel and CI `npm ci` require an exact lockfile match. Keep all `@nx/*` packages on the same version—partial Dependabot bumps desync `package-lock.json` and break `npm ci`. Root `"overrides": { "postcss": "^8.5.10" }` forces patched PostCSS because Next 16.2.x pins vulnerable `8.4.31` until a stable release bundles ≥8.5.10. `lint-staged` runs ESLint/Jest only on `apps/**` and `packages/**` (excludes `docs/` prototypes). `.env.local` stays gitignored; `.env.example` is the committed template for required `NEXT_PUBLIC_*` and related vars. ## Design Context

Strategic product context for the **MediaBubble Launcher** lives in [`PRODUCT.md`](PRODUCT.md) (root); the visual system lives in [`docs/brand/DESIGN.md`](docs/brand/DESIGN.md). Read both before design or UI work so output stays on-brand.

- **Register:** product (app UI; design serves the workflow, not the other way around).
- **Users:** the ~25-person MediaBubble agency, mixed roles and technical comfort; the launcher is their daily driver. Clients see only `/portal`.
- **Brand:** "Obsidian Creative Studio" — confident, crafted, calm. Deep charcoal canvas (#0D0F12), brand-blue (#2196F3) working accent, brand-yellow (#FFC107) reserved for real CTAs, **flat elevation (borders over shadows).**
- **Anti-references:** enterprise/corporate heaviness, consumer-playful/gamified-cute, over-designed/flashy. Craft should be invisible until needed.
- **Accessibility bar:** WCAG 2.1 AA, first-class Arabic/RTL (logical properties, Cairo typeface), and status never by hue alone.
- **Principles:** daily-driver over showpiece; one home for mixed roles; flat elevation + earned motion; invisible craft; auditable and honest (real states, no fake metrics).

The `impeccable` design skill (`$impeccable <command>`) reads `PRODUCT.md` + `DESIGN.md` automatically.
