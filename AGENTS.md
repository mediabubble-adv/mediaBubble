## Learned User Preferences

- Advance phased implementation work with `next` (or equivalent) rather than re-scoping or re-planning unless the user asks to change direction.
- When executing an attached implementation plan, do not edit the plan file; mark existing plan todos in progress and complete them.
- UI change requests often include a browser DOM path and visual requirements—locate the matching component or section in code before editing.
- For frontend polish and section redesigns, follow attached design skills (`impeccable`, `design-taste-frontend`) when the user invokes them.
- Cross-check external audit or bug reports against the current repo layout before implementing fixes—refactors may already have addressed reported items.
- Apply market-site UI and localization work to both `apps/web-eg` and `apps/web-ae` unless one market is specified; keep Egypt (Masri) and UAE (Khaliji) copy distinct between apps.
- Do not run `scripts/clarify-marketing-ae.mjs` on finalized AE Khaliji locale files—it Masri-swaps EG templates; for targeted Khaliji register fixes on AE `ar` locales use `scripts/apply-khaliji-ae-ar.mjs` instead.
- Prefer shorter CTA button and headline copy in English and Arabic—avoid long marketing CTAs.
- Avoid duplicating content blocks across pages (e.g. shared "How we work" on home and about); each service page should have unique sections, blocks, and SEO-aware copy with EG and UAE differentiated.
- Keep `apps/brand` light/dark theme behavior and semantic surface tokens aligned with `web-eg`/`web-ae` when changing shared theme infrastructure or brand CSS variables.
- Brand guidelines search should surface individual items (logo variants, palette swatches, typefaces, components, icons), not only top-level section or category names.
- Refer to the internal ops app as **MediaBubble Launcher** (`apps/launcher`) in user-facing docs and README, alongside MediaBubble Egypt/UAE/Brand product naming.

## Learned Workspace Facts

- MediaBubble is an Nx monorepo: market apps `apps/web-eg` (Egypt) and `apps/web-ae` (UAE), plus `apps/brand` and internal ops app `apps/launcher` (port 3003, `pnpm run dev:launcher`); shared packages include `packages/shared` and `packages/design-system`.
- Shared CSP/security headers and middleware helpers live in `packages/shared/*.cjs` wired from market apps' and brand's `middleware.ts` and `next.config.js`; import CSP via `@mediabubble/shared/csp-middleware`. **MediaBubble Launcher** uses `proxy.ts` for JWT route gating, not `middleware.ts` (Next.js 16 builder). `export const config.matcher` must be an inlined literal array in each `middleware.ts` (constants imported from `.cjs` cannot be statically analyzed and break dev bundles). App `tsconfig.json` files that override `paths` must repeat shared subpath aliases from `tsconfig.base.json` or typecheck fails in middleware. Cross-Nx-project imports (e.g. Tailwind `mbPreset`) must use package subpaths like `@mediabubble/design-system/tailwind-preset`, not relative `packages/` paths—required for `enforce-module-boundaries` lint. `getCspNonce()` in `packages/shared/src/csp-nonce.ts` is async—root layouts in `web-eg`, `web-ae`, and `brand` must `await` it because Next `headers()` is Promise-based in current types/CI.
- Dark mode is class-based (`html.dark`) via shared `ThemeProvider` from `@mediabubble/shared/client` with storage key `mediabubble-theme`; inject `THEME_INIT_SCRIPT` in each app layout `<head>` to prevent flash. `web-eg`, `web-ae`, and `apps/brand` share this stack; `data-theme` is not used.
- Semantic `brand-*` tokens (`bg-brand-surface`, `text-brand-text-muted`, etc.) map to CSS variables in each app's `globals.css` and support `dark:` variants; fixed hex tokens (navy, yellow, blue, red) stay constant across themes.
- Root `package.json` exposes `pnpm run dev:brand` (port 3002), `dev:eg:clean`, `dev:ae:clean`, and `dev:launcher:clean` (port 3003) to reset dev (kill port, wipe `.next`/cache, stale PWA workers on market apps, restart); use after webpack `Cannot read properties of undefined (reading 'call')`, Playwright leaving a stale launcher server on :3003, or overlapping prod builds—avoid production builds while dev is running. In the browser: hard refresh, Incognito, or clear site data for `localhost` if the error persists after a clean restart. Local dev uses root `pnpm` only—do not run `npm install` inside `apps/*` (creates stray per-app `node_modules` and `package-lock.json`); pnpm expects root `pnpm-workspace.yaml` (`packages: apps/*`, `packages/*`)—root `package.json` `workspaces` remains for npm CI. CI installs with `npm ci` on Node 22; `nx.json` sets `cli.packageManager: "npm"` because committed `pnpm-lock.yaml` makes Nx auto-detect pnpm and fail without `node_modules/.modules.yaml`—CI also drops `pnpm-lock.yaml` in the runner as a guard. Keep all `@nx/*` packages on the same version—partial Dependabot bumps desync `package-lock.json` and break `npm ci`. `.env.local` stays gitignored; `.env.example` is the committed template for required `NEXT_PUBLIC_*` and related vars. Launcher env lives in `apps/launcher/.env.local`; root `db:generate`/`db:migrate`/`db:deploy`/`db:studio`/`db:seed` source `.env.local` then `prisma/.env` when present before Prisma (CLI does not load Next.js env paths)—mirror vars in `apps/launcher/prisma/.env` for ad-hoc Prisma CLI. Vercel launcher deploy uses `apps/launcher/vercel.json` → `vercel-build:launcher` (migrate + generate + `nx build launcher`); `db:deploy:ci` falls back `DIRECT_URL=${DIRECT_URL:-$DATABASE_URL}`.
- In `'use client'` files, import browser hooks and tracking from `@mediabubble/shared/client` (not the root `@mediabubble/shared` barrel, which can pull HubSpot/Resend/rate-limit into the client bundle and trigger webpack `.call` errors). Import marketing kicker classes from `@mediabubble/shared/ui/marketing-kicker` when `optimizePackageImports` is enabled. Server Components and API routes should use `@mediabubble/shared/server` or package subpaths.
- Live service slugs in each app's `services-data.ts` are five (`seo`, `ppc`, `social`, `branding`, `web`); `content`/`events` are planned in `docs/content/service-component-inventory.md` until routes land. Run `pnpm run check:i18n` before merge to verify EG/AE locale key parity.
- Arabic content skills live in `.claude/skills/arabic-*`; Cursor loads them via `.cursor/skills/` symlinks. Re-sync with `bash scripts/sync-cursor-arabic-skills.sh`. Pipeline: strategist → creator → `arabic-qa`. Market apps use client-side i18n via shared `useI18n()` from `@mediabubble/shared/client` (not next-intl): per-app `lib/i18n/*.json` merged with `public/locales/*/translation.json`; `web-eg` uses `ar-masri`, `web-ae` uses `ar` with `ar-khaliji.json`.
- RTL layouts: infinite-scroll marquees keep the animation track `dir="ltr"` for seamless `translateX` loop math; set per-card `dir` for Arabic text. Footer and contact phone use `tel:` from `resolveMarketSiteConfig().phone` (ASCII E.164) with `dir="ltr"`, `tabular-nums`, and `unicode-bidi: isolate`; keep Western digits in `ar` locale phone strings. Home hero is full viewport height on both market sites; inner pages use the shorter contact-page `PageHero` height (not full screen).
- Next.js 16 App Router segment config (`export const revalidate`, `runtime`, etc.) must be compile-time literals—do not assign imported constants (e.g. `ISR_REVALIDATE_SECONDS.*`). Dynamic `opengraph-image.tsx` routes with `generateStaticParams` must not set `export const runtime = 'edge'`. All four Next apps (`launcher`, `brand`, `web-eg`, `web-ae`) export plain `next.config.js` without deprecated `@nx/next` `composePlugins()`/`withNx()`—Nx infers via `@nx/next/plugin` in `nx.json`. `web-eg`/`web-ae` `project.json` build targets use `next build --webpack` (PWA/webpack config fails under Next 16 Turbopack default). `apps/brand` client i18n `t()` does not accept interpolation params—substitute dynamic values in components (e.g. `.replace('{{year}}', String(year))`).
