## MODULE 3.1: ONBOARDING — Development Fundamentals (New Hires)

**TARGET ROLE:** All Dev Specialists (Level 1→2)  
**DURATION:** 4 weeks (longer than design due to tech complexity)  
**DELIVERY:** Hybrid (instructor-led + hands-on pairing)

### Learning Outcomes

- Understand MediaBubble's tech stack and architectural philosophy
- Set up development environment (local + cloud infrastructure)
- Proficiency with version control (Git), CI/CD pipelines, testing frameworks
- Ability to complete first ticket end-to-end with code review approval
- Familiarity with code quality standards and QA process

### Curriculum

**Week 1: Development Environment & Fundamentals**

- Day 1–2: Tech stack overview (languages, frameworks, databases, cloud platform)
- Day 2–3: Local development setup (repo cloning, dependencies, build process, environment variables)
- Day 3–4: Git mastery (branching strategy, commit conventions, pull request process)
- Day 5: Architecture walkthrough and codebase orientation
- **Deliverable:** Successful local environment setup + first PR submitted (code review feedback)

**Week 2: Core Development Skills & QA Standards**

- **Backend Team:** API design principles, database design, authentication/authorization, testing (unit, integration)
- **Frontend Team:** Component architecture, state management, responsive design implementation, testing
- **Full Stack Team:** End-to-end development flow, API-UI integration, full test coverage
- **DevOps Team:** Infrastructure as Code, containerization (Docker), Kubernetes basics, monitoring
- **Mobile Team:** Platform-specific setup (iOS/Android), emulator/device debugging, mobile-specific testing
- **QA Team:** Manual testing strategies, automated test frameworks, bug reporting, regression testing
- **Deliverable:** 2–3 simple tickets completed and merged to main branch

**Week 3: Code Review & Quality Standards**

- Pair programming with assigned mentor (2–3 days)
- Code review participation (observing and receiving feedback)
- Testing standards deep-dive (required test coverage %, test types)
- Documentation expectations (comments, README, API docs)
- **Deliverable:** Complete ticket with full test coverage and documentation

**Week 4: Real Project Integration**

- Join real project team
- Complete 2–3 tickets independently (with mentor code review)
- Participate in standup and team ceremonies
- Week 4 evaluation with Team Manager
- **Deliverable:** Fully approved, merged commits demonstrating readiness

### Assessment

- **Practical:** 5 tickets completed with ≥80% approval rate
- **Code Quality:** Code review feedback shows improvement across submissions
- **Test Coverage:** Meeting project minimum coverage requirements
- **Collaboration:** Positive team feedback on communication and attitude

### Success Metrics

- Onboarding completion: ≤4 weeks
- First PR approval rate: ≥75%
- Code coverage on contributions: ≥80%
- Team mentor satisfaction: ≥4.5/5

### Prerequisites

- Programming fundamentals (loops, data structures, OOP)
- Familiarity with at least one language in stack
- Git basics (clone, commit, push)
- Understanding of HTTP/REST (for backend/frontend)

### Resources Required

- Local dev environment setup documentation
- Code review rubric and standards document
- Onboarding codebase (simplified, well-commented)
- 40 hours of pair programming with senior developer
- Access to staging/test environments
